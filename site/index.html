Know which agent needs you.

Claude Mate is a tiny USB desk device that cuts the cognitive overload of orchestrating many Claude Code sessions at once — across different accounts and projects. One glance tells you which session finished, is blocked, or errored; one button takes you straight to it.

The problem it solves

Running a fleet of agents means a wall of terminals, each on its own schedule. The tax isn't the work — it's watching for which one stalled. Claude Mate moves that watching off your screen and onto one calm surface.

🗂️

One triage queue

Every session in a stable, alphabetical order — tabs never shuffle under you. Different accounts and projects, one list.

💡

An LED that means it

A status-distinct blink for the worst unacknowledged alert — finished, waiting, or errored — looping until you deal with it.

⏎

One button, there

Press GO and the exact terminal for the session on screen is raised. No hunting across windows.

What's on the glass

A 0.91″ OLED renders one pre-composed screen — the selected session over a whole-fleet strip. Three buttons, one meaning, always: PREV · GO · NEXT.

LED — START · INPUT · DONE · ERROR, looping until you ack

Fed by the Claude Code hooks or the PTY wrapper, which reads live TUI state and raises the exact terminal on GO.

Parts

A handful of common components. Full pinout and notes in docs/WIRING.md; the bill of materials lives in the README.

  • 1×Arduino Nano (ATmega328P) — any USB-serial clone works
  • 1×SSD1306 0.91″ 128×32 OLED, I²C — addr 0x3C; 0.96″ 128×64 also works
  • 3×Momentary push buttons — PREV · GO · NEXT (INPUT_PULLUP)
  • 1×Indication LED + ~220 Ω–1 kΩ resistor — on D8, the sole alert output
  • 1×USB cable — data-capable, to the Mac
  • —3D-printed enclosure — printable model below

Software: a Python daemon (pyserial) on macOS + the firmware’s only library dependency, Adafruit GFX (the SSD1306 is driven by a bundled software-I²C driver).

Roadmap

Built in iterations, each removing more friction than the last. The goal never changes — one calm surface that tells you which agent needs you.

Iteration 1 · now

Arduino + macOS

An Arduino Nano drives the OLED, buttons, and LED over USB serial; a Python daemon on the Mac keeps the queue, renders the screen, and raises windows.

Iteration 2 · next

ESP32 Wi-Fi remote

Move to an ESP32 so the device talks to the daemon over Wi-Fi instead of USB — a truly wireless desk remote you can place anywhere, no cable to the Mac.
